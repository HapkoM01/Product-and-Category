# Домашнее задание Product and Category

## Описание проекта

Проект для работы с классами Product (Товар) и Category (Категория товаров). Реализован полный функционал для управления
товарами и категориями с защитой данных и магическими методами.

## Реализованный функционал

### Домашнее задание 14.1 (Базовые классы)

- Класс `Product` с атрибутами: name, description, price, quantity
- Класс `Category` с атрибутами: name, description, products
- Атрибуты класса `Category`: `category_count` и `product_count`
- Автоматический подсчет количества категорий и товаров

### Домашнее задание 14.2 (Инкапсуляция)

- Приватный атрибут `__products` в классе `Category`
- Метод `add_product()` для добавления товаров в категорию
- Геттер `@property products` с форматированным выводом товаров
- Приватный атрибут `__price` в классе `Product`
- Геттер и сеттер для `price` с проверкой на положительное значение
- Класс-метод `new_product()` для создания продукта из словаря

### Домашнее задание 15.1 (Магические методы)

- `__str__` для `Product`: "Название продукта, X руб. Остаток: X шт."
- `__str__` для `Category`: "Название категории, количество продуктов: X шт."
- `__add__` для `Product`: сложение по формуле цена × количество
- Оптимизирован геттер `products` с использованием `__str__`

### Дополнительное задание

- Класс `CategoryIterator` для перебора товаров в категории
- Поддержка итерации в цикле `for`

### Домашнее задание 16.1 (Наследование)

### Класс Smartphone (наследник Product)

- Дополнительные атрибуты: efficiency, model, memory, color
- Переопределен `__str__` с дополнительной информацией
- Наследует все методы Product

### Класс LawnGrass (наследник Product)

- Дополнительные атрибуты: country, germination_period, color
- Переопределен `__str__` с дополнительной информацией
- Наследует все методы Product

### Ограничения сложения

- Проверка типов при сложении через `type()`
- Ошибка `TypeError` при сложении разных классов

### Ограничения добавления в категорию

- Проверка через `isinstance()` в методе `add_product()`
- Ошибка `TypeError` при добавлении не-продукта

### Домашнее задание 16.2 (Абстрактные классы и миксины)

- **Абстрактный класс `BaseProduct`**:
    - Определяет общий интерфейс для всех продуктов
    - Содержит абстрактные методы: `__init__`, `get_info`, `__str__`, `__add__`
    - Все классы продуктов наследуются от `BaseProduct` через `Product`

- **Класс-миксин `ReprMixin`**:
    - При создании объекта выводит информацию о классе и параметрах
    - Пример: `Product('Samsung Galaxy S23 Ultra', '256GB...', 180000.0, 5)`
    - Реализует метод `__repr__` для красивого представления объекта
    - Добавлен в цепочку наследования класса `Product`

### Домашнее задание 17.1 (Обработка исключений)

#### Проверка количества товара

- При создании продукта с нулевым или отрицательным количеством выбрасывается исключение `ValueError`
- Сообщение: "Товар с нулевым количеством не может быть добавлен"
- Проверка работает для всех классов-наследников (Smartphone, LawnGrass)

#### Метод среднего ценника

- Реализован метод `middle_price()` в классе `Category`
- Складывает цены всех товаров и делит на количество товаров
- Обработка `ZeroDivisionError`: при пустой категории возвращает 0
- Использование конструкции `try/except` для безопасного вычисления
